# Feel

## Primary quality target
Quattro must feel precise, readable and fair.

## What “good feel” means here
- no ambiguous input response
- stable pacing
- readable board under pressure
- consistent rotation behavior
- lateral movement that feels deliberate and controllable
- difficulty that ramps firmly but not arbitrarily

For the current host-side spike baseline and the documented provisional feel decisions, see `docs/notes/host-feel-baseline.md`.

## Tuning notes to track
- spawn timing
- gravity curve
- repeat behavior / input repeat timing
- lock behavior
- readability of occupied vs active cells
- visual feedback on line clear

## Anti-goals
- flashy feedback that hurts readability
- sluggishness disguised as weight
- overly modern convenience features
