# MVP

## Functional MVP
- Boot to title screen
- Start a run
- Spawn pieces correctly
- Move left/right
- Rotate
- Fall over time
- Soft drop
- Lock to board
- Clear lines
- Increase score
- Reach game over
- Restart cleanly

## Technical MVP
- Reproducible build
- Emulator run command
- Basic source tree separation
- At least minimal validation of core rules
- Stable enough for repeated playtests

## MVP exit criteria
The MVP is complete when the project proves its thesis:
- the core loop works
- the control feel is promising
- the codebase direction is sustainable
